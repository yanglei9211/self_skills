整理自 v1 实战发现的限制 + 4-30 小米实战翻车复盘。
按"必做（高价值/低成本）→ 选做（高价值/高成本）→ 可选"分级。

## 🟥 P0 — 必做（v1 已证明的硬伤，不修不舒服）

### P0-1. 港股年报 PDF 解析 (CID 编码 fallback)
- **背景**：4-30 拉小米 2025 年报（415 页 PDF），pdfplumber 抽出来全是 `(cid:14062)` 字符
- **原因**：港股年报多用图片化的繁体字字体，PDF 内嵌字体没有 unicode 映射
- **方案**：`pdf_extract.py` 加一层 fallback —— 检测到 cid 字符占比 > 50% 时自动降级到 OCR
  - 用 `pdf2image` 把每页转 PNG（依赖 poppler）
  - 用 `pytesseract` + `chi_tra`（繁体）/ `chi_sim`（简体）做 OCR
  - 缓存 OCR 结果（OCR 慢，年报基本不变）
- **工作量**：~2 小时
- **难点**：tesseract 需要本地安装，繁体识别准确率约 90%

### P0-2. 风险扫描 ST 误报修复
- **背景**：`risk_scan.py` 对雪球 `market=st` 的返回直接打 ST_NAME 标签，但雪球该分类宽泛地包含一些"风险警示"股票（如太极股份、中创智领），实际并非 ST 股
- **方案**：在 `rule_st_stocks` 里加 `if "ST" in name.upper() or "*ST" in name`，name 不含 ST 关键字的不打 ST_NAME 标签，改打 `RISK_BOARD`
- **工作量**：~10 分钟

### P0-3. 同花顺概念归属抽错
- **背景**：`get_a_concepts(code)` 返回的是其他公司名（如宁德时代返回"丰元股份/天赐材料"），不是真正的概念名（"动力电池/储能"）
- **原因**：xpath `//div[@class='gnContent']//a/text()` 抓的是同概念其他成分股的链接文本，不是概念名标题
- **方案**：换 xpath 到 `//div[@class='m_box gnList']/h3` 或者 `<table id="thsConcept">` 抓表头
- **工作量**：~30 分钟（需要重新抓页面看真实结构）

### P0-4. 港股公司基本信息维度补全
- **背景**：港股 `get_hk_company_info(code)` 只返回 3 项（代码/现价/市值），缺英文名/上市日期/主营业务/董事长等
- **方案**：接 `https://stock.finance.sina.com.cn/hkstock/company/<code>.html` 解析公司资料表
- **工作量**：~45 分钟（需要先看新浪港股页面 HTML 结构）

### P0-5. fetch_market_news.py 的 sina 源彻底修复
- **背景**：v1 修了一半，新浪 lid=2509/2510 能拿到数据但**内容混杂**（财经+体育+社会都有）
- **方案**：拉到的每条新闻先按"标题/正文含财经关键词"过滤再返回
  - 关键词：股市/A 股/港股/美股/降息/通胀/财报/市值/IPO/央行/监管/concentr/Fed/inflation/earnings 等
- **工作量**：~30 分钟

## 🟧 P1 — 高价值，应做（用户体验/数据完整性）

### P1-1. 同业横向对比的财务维度
- **背景**：`supply_chain.py` 的 peers 当前只返回行情（市值/涨跌/成交），缺 PE/PB/ROE/营收增速等核心财务对比
- **方案**：peers 里改成调雪球 screener，单只单只拿全字段（screener 支持 sym_filter）
- **工作量**：~1 小时

### P1-2. 雪球热门帖（散户情绪）启用
- **背景**：v1 因阿里云 WAF 关闭了雪球 hot/listV2.json
- **方案**：增加配置入口
  - 用户在 `~/.config/stock-market-hub/xueqiu.cookie` 写入 Chrome 拷贝来的 cookie
  - `core/xueqiu.py` 启动时读这个文件，有 cookie 就启用 hot/comments 接口
- **工作量**：~1 小时
- **额外收益**：能拿到散户帖子情绪、个股评论、大 V 观点

### P1-3. 中概股 SEC EDGAR 集成
- **背景**：BABA / JD / PDD / BIDU 等中概股目前只能拿行情，没有财报/公告/治理结构
- **方案**：新增 `core/sec_edgar.py`
  - SEC EDGAR 是 SEC 官方免费 API（`https://data.sec.gov/submissions/CIK<10位编号>.json`）
  - 通过 ticker → CIK 编号映射 + 拿 10-K / 10-Q / 8-K 公告
  - `analyze_company.py` 美股分支接入
- **工作量**：~3 小时
- **特别提示**：SEC EDGAR 要求 User-Agent 包含 email（合规要求）

### P1-4. 股价相关重磅事件时间轴
- **背景**：用户问"小米最近为什么大跌"，目前数据零散在多个脚本里
- **方案**：新增 `event_timeline.py`，把以下事件按时间线整合：
  - 公告（巨潮 / 披露易）
  - 大跌日（K 线 percent ≤ -5%）
  - 重要新闻（财联社 + 海外财经 + 命中关键词）
  - 财报披露日
- **工作量**：~2 小时
- **价值**：用户问"为什么大跌"时一表呈现因果关系

### P1-5. CLI 入口统一封装
- **背景**：当前 8 个脚本独立调用，agent 要记住一堆参数。一个统一的 `smh` 命令更方便
- **方案**：写 `bin/smh.py`，子命令模式：
  - `smh company SZ300750 --deep`
  - `smh sector "AI PC"`
  - `smh news --today`
  - `smh risk --rules R1,R2,R5`
  - `smh announcements SZ300750 --keyword "立案"`
- **工作量**：~1 小时（argparse 子命令包装）
- **额外**：在 SKILL.md 里把所有调用例子统一改成 `smh xxx`

### P1-6. 增量缓存层
- **背景**：同股票多次查询会重复抓取，浪费时间和限频额度
- **方案**：`core/cache.py`，按 (function_name, args) 哈希缓存到 `~/.cache/stock-market-hub/data/`
  - 行情类 TTL=60s
  - 公告类 TTL=4h
  - 公司基本信息 TTL=24h
  - K 线 TTL=4h（盘后），盘中 TTL=60s
  - 财报 PDF 永久缓存
- **工作量**：~1.5 小时

## 🟨 P2 — 选做（高价值但成本/复杂度较高）

### P2-1. 上下游图谱用 LLM 抽取
- **背景**：`supply_chain.py` 当前用纯正则启发式抽取客户/供应商，精度有限
- **方案**：把 PDF 章节文本传给 LLM（Claude/GPT）做语义抽取
  - 输入：年报"主要客户"/"主要供应商"章节文本
  - 输出：JSON Schema `[{name, type, ratio, business}]`
  - 再调 LLM 一次：基于业务概要生成"商业模式 1 句话总结"
- **工作量**：~3 小时
- **难点**：需要 API key 配置 + 成本预估

### P2-2. 卖方研报观点抓取
- **背景**：v1 完全没研报数据
- **方案**（免费源）：
  - 东财研报频道：`https://data.eastmoney.com/report/stock.jshtml?stockcode=300750`（HTML 解析）
  - 同花顺免费摘要：`https://stock.10jqka.com.cn/report/300750_yqgg.shtml`
  - 注意：完整研报 PDF 通常付费墙，但摘要 + 评级 + 目标价免费
- **方案**（付费源）：
  - Tushare Pro 200-500 元/年提供研报评级数据
- **工作量**：~3 小时（仅免费摘要）

### P2-3. 大股东持股质押 / 减持监控
- **背景**：v1 风险扫描里 R8（减持）只是关键词搜索，没结构化
- **方案**：新增 `monitor_pledge.py`
  - 巨潮专项数据：`http://www.cninfo.com.cn/data20/...`（股东质押）
  - 雪球 screener 有质押率字段（`pledge_rate`）
- **工作量**：~2 小时

### P2-4. 跨市场同公司联动（A+H / ADR / 中概双重上市）
- **背景**：腾讯只有港股、阿里同时港股+美股、宁德时代 A+H、比亚迪 A+H
- **方案**：维护一个 `data/cross_listing.json` 映射表，分析单一公司时自动拉 A+H 双行情
- **工作量**：~2 小时（手工整理映射表）

### P2-5. 行业景气度指标
- **背景**：分析单只股票时缺"行业基本面"维度
- **方案**：每个行业整合 1-2 个高频指标
  - 半导体：费城半导体指数 / WSTS 月度销售
  - 新能车：中汽协月度销量
  - 房地产：30 城新房成交、土拍
  - 白酒：飞天茅台批价（雪球能爬）
- **工作量**：~3 小时（每个行业的源不同）

## 🚦 当前 v1 已知限制（用户需要知道的）

| 维度 | 限制 | Workaround |
|---|---|---|
| 港股年报 PDF | CID 编码无法直接解析 | 用户自己打开 PDF 看摘要 |
| 雪球热门帖 | WAF 拦截，匿名访问被拒 | v2 配置 cookie |

| 披露易接口 | 偶尔超时 | 已加 3 次重试，单源失败不影响其他 |
| 实时盘中行情 | 雪球/腾讯延迟 0-60s | 高频策略不要用 |

## 📋 实战错误记录（用于持续改进约束规则）

每次实战发现的"凭印象出错"案例，立即记录在此，同步补充到 SKILL.md 硬约束。

| 日期 | 错误描述 | 错误原因 | 修复方式 |
|---|---|---|---|
| 2026-04-30 | 写"HK$ 28（年内多次震荡支撑位）" | 没调 K 线，凭印象 | 调 `price_history.thresholds` → 28 是当天才第一次跌破（514天前才有历史） |
| 2026-04-30 | 写"6-7 月 YU7 正式发布（最大催化剂）" | 没读年报，凭训练集记忆 | 读年报 p.7 → YU7 已连续销售 7 个月，3 月新一代 SU7 才是最新动作 |
| 2026-04-30 | 写"原油涨价→小米上游成本压力" | 因果链推理错误 | 原油涨对电动车是利好（加速替代），对小米供应链影响极小；且价格数据需来自新闻源 |
| 2026-04-30 | 把 4-28"年度報告"说成"年报首次发布，引发信息消化期" | 没查公告时间线 | 调 `fetch_announcements --days 90` → 真正有信息量的业绩公告是 3-24 发布的，4-28 只是正式装订版 |

### 根本原因

以上 4 个错误有共同特征：**工具已经配好了，但没有先跑工具，直接开始写分析**。

解决方案已写入 SKILL.md §3"核心约束"，强制要求"分析前必须做的动作"：
1. `fetch_announcements` 先看公告时间线
2. `pdf_extract` 读原文数字
3. `fetch_market_news` 找宏观新闻来源
4. `price_history` 做价位判断
5. `analyze_company` 整合产出
